# 1.1 显式余弦检索

## 为什么升级

1.0 已经使用同一个 `SentenceTransformer` 模型为资料片段和查询生成归一化向量，但 collection 创建时没有声明距离空间。Chroma 的默认距离是 squared L2：

```text
d_l2 = Σ(Ai - Bi)²
```

单位向量上有 `d_l2 = 2 - 2 × cosine_similarity`，因此 L2 与余弦通常给出相同排序；但 L2 distance 不是余弦距离，旧代码的 `1 / (1 + distance)` 也只能称为相关度转换分数，不能称为真实余弦相似度。

1.1 新建独立 collection，并通过 Chroma 1.5.9 的配置 API 显式声明：

```python
client.get_or_create_collection(
    name="course_material_chunks_v1_1_cosine",
    configuration={"hnsw": {"space": "cosine"}},
)
```

Chroma 对 cosine 空间返回的定义是：

```text
distance = 1 - cosine_similarity
similarity_score = 1 - distance
```

`similarity_score` 在浮点误差截断后的范围为 `[-1, 1]`。前端百分比使用 `max(0, similarity_score) × 100`，不会把 distance 当作百分比。

参考：[Chroma collection 配置](https://docs.trychroma.com/docs/collections/configure)。

## 统一检索流程

普通检索接口、Agent 预检索和 Agent 概念工具全部调用：

```python
retrieve_course_chunks(
    db,
    user_id=user_id,
    course_id=course_id,
    query=query,
    top_k=5,
    min_similarity=None,
)
```

服务按以下顺序工作：

1. 校验问题长度、Top-K 和阈值。
2. 用 `Course.id + Course.user_id` 验证课程归属。
3. 可选资料筛选逐项验证 `Material.id + user_id + course_id`。
4. 使用同一模型及 `normalize_embeddings=True` 生成查询向量。
5. Chroma where 同时限定 `user_id + course_id`。
6. 查询最多 `min(top_k × 3, 50)` 个候选。
7. 使用 `similarity_score = clamp(1 - distance, -1, 1)`。
8. 丢弃低于阈值的候选。
9. 从 MySQL 联表二次验证 `MaterialChunk` 和 `Material` 的用户、课程及关联关系。
10. 以 MySQL 的 `MaterialChunk.content` 为正文，按 chunk 去重并保留 Chroma 顺序。

Chroma metadata 与 MySQL 不一致时，该候选会被丢弃并记录不含正文的 warning，不会交给用户或 Agent。

## 配置

```dotenv
CHROMA_PATH=./chroma_db
CHROMA_COLLECTION_NAME=course_material_chunks_v1_1_cosine
SEMANTIC_SEARCH_TOP_K=5
SEMANTIC_SEARCH_MAX_TOP_K=20
SEMANTIC_SEARCH_MIN_SIMILARITY=0.35
EMBEDDING_MODEL_NAME=sentence-transformers/paraphrase-multilingual-MiniLM-L12-v2
EMBEDDING_NORMALIZE=true
EMBEDDING_BATCH_SIZE=16
```

兼容旧变量 `CHROMA_COLLECTION`、`EMBEDDING_MODEL` 和 `RAG_TOP_K`。若旧变量仍指向 L2 collection，服务不会修改它，而是明确拒绝并提示执行迁移。1.1 的 cosine collection 强制 `EMBEDDING_NORMALIZE=true`。

默认阈值 `0.35` 是保守初值，不代表所有课程的最佳阈值。应使用真实课程问答集统计“相关片段召回率”和“无关片段拒绝率”，再在 `[-1, 1]` 范围内调优。阈值过高会漏掉表述差异大的依据，过低会增加无关上下文。

## API

原接口保持不变：

```http
POST /api/search/semantic
```

新增响应字段：

- `metric: "cosine"`
- `min_similarity`
- `course_id`、`course_name`
- `file_type`
- `similarity_percent`

兼容字段 `vector_id`、`distance`、`similarity_score` 继续保留，但含义已明确：distance 越小越相关，similarity_score 越大越相关。

没有候选达到阈值时返回空 `results`，Agent 会说明“当前课程资料中未检索到相似度足够高的内容”，不会为了填满 Top-K 使用低质量片段。

## 限制

- 语义相关不等于事实必然正确，用户仍应展开引用核对原文。
- PDF 文本层丢失的字符不能靠向量检索恢复，应通过 PyMuPDF/OCR 改善解析。
- 近似最近邻检索可能受 HNSW 参数和数据规模影响；当前保持 Chroma 默认的其他 HNSW 参数。
