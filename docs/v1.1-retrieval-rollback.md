# 1.1 检索回滚

## 运行配置回滚

停止应用后，把本地 `.env` 的 collection 配置切回迁移前名称：

```dotenv
CHROMA_COLLECTION_NAME=course_material_chunks
```

然后切换到兼容 1.0 L2 语义的代码版本。1.1 代码会拒绝把 L2 collection 当成 cosine 使用，这是防止误读 distance 的保护行为。

切换配置不会删除新旧任何 collection。旧 collection 没有被迁移脚本更新，因此能恢复迁移前的索引视图。

## 代码回滚

```powershell
git switch release/1.1.0
```

需要核对原始 1.0 时，可从只读恢复标签新建排障分支：

```powershell
git switch -c recovery/v1.0-retrieval v1.0.0-retrieval-backup
```

不要在有未提交修改时切换分支；先提交或 stash。

## 新增资料说明

1.1 运行期间新增资料默认只进入 1.1 cosine collection。代码回滚后若旧系统必须检索这些新增资料，需要在旧环境重新建立向量。旧 collection 被保留不等于它自动获得 1.1 的增量数据。

## 可选清理新 collection

普通回滚不需要删除新 collection。只有完成 MySQL 与 `chroma_db` 备份、确认已经切回旧 collection、并明确不再需要 1.1 索引时，管理员才可使用 Chroma 客户端手工删除精确名称 `course_material_chunks_v1_1_cosine`。不要删除整个 `chroma_db`，不要使用模糊名称，也不要把该操作加入启动或回滚脚本。

建议优先保留新 collection，待新版本稳定和备份验证完成后再决定是否清理。
