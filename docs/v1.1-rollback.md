# 1.1 回滚

1. 升级前使用 `mysqldump` 保存 1.0 数据库，并保留 `uploads/`、`chroma_db/` 的本机副本。
2. 代码恢复点为 `v1.0.0-local-backup`；不要在当前工作区执行 hard reset。
3. 推荐把 1.0 备份恢复到新的 MySQL 数据库，再把本机 `DATABASE_URL` 指向该库。
4. 检索回滚只需将 `CHROMA_COLLECTION_NAME` 切回旧 collection；1.1 不删除旧 collection。
5. Alembic downgrade 只移除 1.1 新结构，但会丢失头像、综合规划关联和知识图谱，因此只有已备份时才执行。

```powershell
git switch -c restore/v1.0.0 v1.0.0-local-backup
```

1.1 后新增资料默认只写入新的 cosine collection。回滚后若需检索这些资料，应在旧环境显式重建旧索引。
